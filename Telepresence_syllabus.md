
# Telepresence Lab Syllabus

## Audience
Beginners (no prior experience with Telepresence required)

## Session Overview
**Duration:** 1 session (2-3 hours)
**Goal:** Achieve a practical understanding of Telepresence and its use for local development with remote Kubernetes clusters.

---

## Learning Objectives
- Understand what Telepresence is and why it is useful
- Install and configure Telepresence
- Connect local development environment to a remote Kubernetes cluster
- Intercept and debug services using Telepresence
- Use Telepresence for rapid development workflows
- Troubleshoot common issues

---

## Session Outline

### 1. Introduction to Telepresence (15 min)
	- What is Telepresence?
	- Use cases and benefits
	- Overview of lab goals

### 2. Prerequisites & Setup (20 min)
	- Required tools: kubectl, Docker, Kubernetes cluster access
	- Installing Telepresence
	- Verifying installation

### 3. Connecting to a Remote Cluster (20 min)
	- Authenticating with Kubernetes
	- Running your first Telepresence command
	- Understanding the connection process

### 4. Service Interception & Local Development (40 min)
	- What is service interception?
	- Intercepting a service
	- Redirecting traffic to local code
	- Live coding/demo: Make a change locally and see it reflected in the cluster

### 5. Debugging & Troubleshooting (20 min)
	- Using Telepresence logs
	- Common errors and solutions
	- Q&A

### 6. Wrap-up & Next Steps (5 min)
	- Review key takeaways
	- Additional resources
	- Feedback
